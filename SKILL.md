# XHS Note

## 目标

把用户提供的小红书笔记表格处理成一套可检查的批量生成结果：

1. 筛选内容超过 200 字的笔记。
2. 抽象每条笔记的标题模板、内容模板和选题方向。
3. 按选题方向联网搜索素材。
4. 倒序套用模板，生成适合发小红书的新标题和新内容。
5. 输出筛选表、模板表和最终对比 Excel。

处理表格时优先使用 Spreadsheets skill。涉及搜索、拓展、最新信息或来源链接时必须联网搜索，除非用户明确说不要联网。

## 输入识别

支持 `.xlsx`、`.xls`、`.csv`、`.tsv`。

标题列优先识别：`笔记标题`、`标题`、`title`、`note_title`。

内容列优先识别：`笔记内容`、`内容`、`正文`、`content`、`note_content`。

如果标题列或内容列有多个候选，先用一句话向用户确认，不要猜。所有阶段都必须保留原始行号，方便追溯回原表。

## 流程

### 1. 读取文件

用户提供表格文件，表里应包含标题列和内容列。

阶段产出：识别出的标题列、内容列、输入总行数。

### 2. 筛选内容

对内容单元格做首尾空白清理后计算字符数，只保留内容字符数严格大于 200 的行。

阶段产出：筛选表 `filtered_notes_over_200_chars.xlsx`，包含：

- `原文行号`
- `原标题`
- `原内容`
- `原文字数`

### 3. 抽象模板和选题方向

基于筛选后的每一条笔记，抽象三样东西：

- `标题模板`：保留标题表达结构，把具体实体、数字、结论替换成变量。
- `内容模板`：保留内容组织方式，不忠实复述原文。
- `选题方向`：用一句话说明后续搜索和生成要围绕什么主题。

阶段产出：模板表 `xhs_note_template_list.xlsx`，包含：

- `模板行号`
- `原文行号`
- `原标题`
- `标题模板`
- `内容模板`
- `选题方向`

### 4. 联网搜索并生成新内容

每条原文使用自己的 `选题方向` 联网搜索素材，然后套用倒序分配到的模板，生成新的小红书标题和内容。

倒序模板规则：

```text
假设筛选后共有 N 条：
第 1 条原文使用第 N 个模板
第 2 条原文使用第 N-1 个模板
第 i 条原文使用第 N+1-i 个模板
```

生成要求：

- 新标题和新内容必须中文输出。
- 新内容不少于 400 字，除非用户另有指定。
- 新标题和新内容要适合小红书发布。
- 原文只用于确定选题方向和提炼模板，不作为逐段改写底稿。
- 硬事实、数字、时间、政策、公司信息、市场信息必须来自联网搜索结果或写明来源不足。
- 每条结果必须保留来源 URL 或来源备注。
- 每条结果的参考来源必须大于等于 4 条。

阶段产出：每条的 `新生成标题`、`新生成内容`、`参考来源`、`拓展素材摘要`、`备注`。

## 并行规则

如果筛选后笔记数超过 10 条，第 4 步必须使用 sub-agent 并行处理。

主 agent 负责第 1-3 步、拆分行号区间、收集结果和写最终 Excel。sub-agent 只负责自己区间内的“联网搜索 + 生成标题正文 + 返回来源备注”。

并行要求：

- 每个 sub-agent 必须使用 `xhigh` / Extra High 推理强度。
- 每个 sub-agent 只能处理自己负责的原文序号范围。
- 每个 sub-agent 必须返回结构化结果。
- 不得用单进程脚本循环、分块 JSON 或批处理脚本替代 sub-agent 并行。
- 如果当前环境无法启动 sub-agent，必须暂停并向用户说明原因，不能静默降级。
- 收集完结果后，关闭所有已完成的 sub-agent。

## 最终对比表

最终文件默认命名为 `all_notes_original_vs_generated_reverse_templates.xlsx`。

主表 `生成对比` 必须把这 4 个核心字段放在最前面：

- `原文标题`
- `原文内容`
- `新生成标题`
- `新生成内容`

后面可以保留这些辅助校验字段：

- `生成序号`
- `原文行号`
- `使用模板行号`
- `选题方向`
- `原文字数`
- `生成字数`
- `标题模板`
- `内容模板`
- `参考来源`
- `拓展素材摘要`
- `备注`

另建 `生成汇总` 表，记录输入总行数、筛选后行数、生成行数、倒序模板校验结果、空标题/空正文检查、来源不足行。

Excel 要冻结首行、长文本自动换行、列宽方便人工检查。

## 验收

最终回复前必须检查：

- 筛选后行数和生成行数一致。
- 每行都有原文标题、原文内容、新生成标题、新生成内容。
- 倒序关系正确：`生成序号 + 使用模板行号 = N + 1`。
- 每行都有来源 URL 或来源备注。
- 新标题不是原标题简单替换，新内容不是原文逐段同义改写。
- Excel 可以打开，排版可读。
- 如果筛选后超过 10 条，最终回复必须说明拆分了几个 sub-agent 区间；如果没有并行，必须说明被阻止的原因。

## 回复用户

最终回复用中文，保持简洁，包含：

- 处理了多少条，生成了多少条。
- 最终对比 Excel 链接。
- 筛选表和模板表链接。
- 是否有来源不足或异常行。
